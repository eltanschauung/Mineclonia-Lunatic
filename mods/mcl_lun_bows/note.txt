LLMs struggled with this Minetest particle system because it required understanding multiple layers of coordinate system transformations that aren't explicitly documented in a single place. The arrow entity uses a mesh with a built-in -90 degree yaw offset (YAW_OFFSET), which means the arrow's local coordinate system is rotated compared to standard world coordinates—the arrow points in the local -X direction rather than the expected +Z direction. When particles are attached to an entity, their positions and velocities are interpreted in that entity's local space, not world space, a subtle distinction that's easy to miss without hands-on testing. The debug logs showed correct world-space calculations but incorrect visual results, creating a misleading signal that the math was right when the actual problem was coordinate frame confusion. Additionally, LLMs lack the ability to visually observe the particle behavior in-game, making it difficult to diagnose that particles were consistently rotated 90 degrees rather than randomly misplaced. This type of engine-specific implementation detail requires either prior experience with the codebase or iterative testing that LLMs cannot perform independently.
